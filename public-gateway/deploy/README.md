# Public Gateway Deploy Bundle

Managed runtime assets generated by `gateway-admin`.

## Generate runtime files

```bash
cd public-gateway
node ./bin/gateway-admin.mjs init
```

This writes:

- `public-gateway/deploy/runtime/docker-compose.yml`
- `public-gateway/deploy/runtime/.env`
- `public-gateway/deploy/runtime/config.json`
- `public-gateway/deploy/runtime/.env.example`

## Lifecycle

```bash
node ./bin/gateway-admin.mjs deploy up
node ./bin/gateway-admin.mjs deploy status
node ./bin/gateway-admin.mjs deploy logs --service public-gateway
node ./bin/gateway-admin.mjs deploy restart
node ./bin/gateway-admin.mjs deploy down
```

Use `--volumes` with `deploy down` to remove Redis, relay, blind-peer, and proxy data volumes.

## Templates

`public-gateway/deploy/templates/` includes profile templates:

- `docker-compose.local.yml`
- `docker-compose.internet.yml`
- `env.local.example`
- `env.internet.example`

The wizard selects a profile and emits concrete runtime files.
